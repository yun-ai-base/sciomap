## 一、数据层：冗余、断裂与分布问题

### 1.1 线上部署了 663KB 的死文件 ⚠️ P0
- `data/scientists.json`（663KB）被 git 跟踪并部署到 Pages，但 `index.html` 第 8 行只加载 `data/scientists.js`。
- 二者内容几乎一致（仅差 `var _sciomapData =` 包裹）。
- **影响**：每次打开网站多下载 663KB 无用流量，拖慢首屏。
- **建议**：删除 `data/scientists.json`（本地用 gitignore 忽略即可），或反过来把数据文件统一成 `.json` + 用 `fetch` 加载（见 2.1）。

### 1.2 4 条悬空关系（断链，静默丢弃）⚠️ P0
以下关系的 `target` 在科学家列表里不存在，渲染时被 `processData` 跳过，导致"师承/影响树"点不开：
```
einstein  -> minkowski     （闵可夫斯基未录入为节点）
mendel    -> watson-crick  （应为 watson 或 crick，无此组合 id）
ritchie   -> thompson      （肯·汤普森未录入）
torvalds  -> stallman      （斯托曼未录入）
```
- **建议**：要么补录这 4 个节点（闵可夫斯基、沃森/克里克修正、肯·汤普森、斯托曼），要么把 target 改对。这些都是真实且重要的科学关联，丢了可惜。

### 1.3 "影响力"区间被压缩，星图区分度弱 ⚠️ P1
- 实际 `influence` 取值只有 **6–10**，均值 8.58。
- 半径公式 `5 + influence*1.5 + 跨标签加成`（第 480 行），导致所有星球大小几乎一样，"亮度=影响力"的核心隐喻难以体现。
- **建议**：
  - 把影响力拉开到更大的刻度（如 1–10 或 1–5），让爱因斯坦/牛顿/达尔文明显更亮更大；
  - 或改用对数/分档映射，让 top 10 一眼可见。

### 1.4 核心叙事字段严重偏薄（项目灵魂没撑起来）⚠️ P1
方案里"近看是问题链（前人卡在哪→谁突破→吵了什么）"是区别于普通人物图谱的关键，但数据里：
- **瓶颈节点（历史难题问号）只有 3 个** —— 概念核心是"问题链"，现在几乎没内容。
- `problemsSolved` 仅 **34/121（28%）** 有；
- `controversies` 仅 **13/121**，`debate` 连线仅 **5 条** —— "学术交锋"弹窗功能基本用不起来；
- `anecdote` 40/121、`keyWorks` 42/121、`legacy` 42/121、`famousEvents` 70/121。
- **建议（按性价比排序）**：
  1. **补瓶颈节点到 15–25 个**（以太之谜、燃素说、地心说、热寂悖论、紫外线灾难、NP 完全性、蛋白质折叠……），并把 `solvedBy` 串成"问题→突破者"链；
  2. 给 top 40 影响力科学家补全 `problemsSolved` 与 `controversies`；
  3. 让瓶颈节点 hover/点击能高亮"谁解决了它"（目前点击跳转已做，但数量太少看不出效果）。

### 1.5 文化/地域覆盖偏科 ⚠️ P2
- `culturalTag`：western 86 / chinese 33 / **eastern 仅 2**。
- 伊斯兰黄金时代、印度（阿耶波多、拉马努金）、日本（汤川秀树、山中伸弥）几乎缺席，但筛选器和文化标签框架都在，是"有架子没内容"。
- **建议**：补 8–15 位非西方科学家，让"文化"筛选真正有意义。

## 二、前端性能与稳定性

### 2.1 D3 走 CDN，国内首屏不稳 ⚠️ P1
- 第 7 行 `<script src="https://cdn.jsdelivr.net/npm/d3@7">`。
- 你另一个项目 philomap 已经吃过这个亏并改成**自托管 `vendor/d3.min.js`**。sciomap 还没改。
- **建议**：把 d3.min.js 自托管到 `data/d3.min.js` 或 `vendor/d3.min.js`，本地引用。首屏更快、断网/墙内也能开。

### 2.2 背景星空 rAF 永久空转 ⚠️ P1
- `drawBg()`（第 404 行）`requestAnimationFrame` 永不停止，200 颗星 + 8 团星云每帧重绘。
- 切到"孪生宇宙"视图时 `bg-canvas` 设了 `display:none`，但 rAF 仍在后台跑，白白吃 CPU/电量。
- **建议**：
  - 切到孪生宇宙时 `cancelAnimationFrame` 暂停，切回再恢复；
  - 或把帧率降到 ~30fps；
  - 或参考 philomap 做成"低开销 + 可见性门控"。

### 2.3 `applyAllFilters` 里对连线的处理有冗余/潜在 bug ⚠️ P2
- 第 546–555 行对 `gEdges` 跑了**两段几乎重复的循环**：第一段带过渡设 opacity+display，第二段又无条件覆盖 display。
- 第二段会忽略第一段过渡，直接 `display:none/null`，导致筛选动画"跳变"而不是淡出。
- **建议**：删掉第二段循环，让第一段统一处理 opacity 过渡即可。

### 2.4 窗口 resize 后布局不重算 ⚠️ P2
- `svg` 宽高在 `renderGraph` 里只设一次；resize 时星星位置不变，可能跑出可视区。
- **建议**：监听 resize，重设 svg 尺寸并 `simulation.alpha(.3).restart()` 或重算 forceX/forceY 的中心。

### 2.5 线上访问计数器是"假"的 ⚠️ P2
- 第 1170 行 `fetch('http://localhost:3456/hit/sciomap')` —— 只在你本机 stats-server 运行时才有效，部署到 Pages 后 404 静默失败。
- **建议**：要么接一个 serverless 计数（如 GitHub API / 免费计数服务），要么直接删掉这行，别留着没用的代码。

- **标签可读性**：长中文名字号压到 5–7px（第 379/754 行），默认缩放下基本看不清。建议：仅对 `influence≥阈值` 或 hover 时显示名称；或默认只显示姓氏/缩写。

- **头像兜底**：115/121 有 http 头像，部分外链可能 404；当前 `onerror` 已回退到首字母，体验 OK，可保留。
